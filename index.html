المشكلة

الوضع الحالي

رغم التطور التكنولوجي، ما زالت العديد من المدارس تعتمد على الأساليب التقليدية في إدارة العملية التعليمية، مثل تسجيل الحضور يدويًا وضعف التواصل بين الطلاب والمعلمين وأولياء الأمور.

الأسباب

يرجع ذلك إلى الاعتماد على الأنظمة الورقية وعدم وجود منصة موحدة، مما يؤدي إلى انخفاض كفاءة الإدارة، وضعف متابعة الطلاب، وإهدار الوقت والجهد، بالإضافة إلى عدم دقة بعض البيانات.

الحاجة

لذلك، ظهرت الحاجة إلى نظام إلكتروني متكامل يساعد في تحسين جودة التعليم وتنظيم العملية التعليمية بشكل أكثر كفاءة.

Data Analysis

تحليل البيانات

L0

DFE LEVEL 0

يوضح هذا المستوى النظام كعملية رئيسية واحدة، مع توضيح تفاعل المستخدمين الخارجيين مثل الطالب والمعلم وولي الأمر والأدمن وتدفق البيانات بينهم وبين النظام.

L1

DFD Level 1

يوضح هذا المستوى تقسيم النظام إلى عمليات فرعية مثل تسجيل الدخول، إدارة بيانات الطلاب، تسجيل الحضور بالذكاء الاصطناعي، إدارة النتائج، وقواعد البيانات المرتبطة بكل جزء.

ERD

يوضح مخطط الـ ERD الكيانات الأساسية في النظام مثل الطلاب والمعلمين وأولياء الأمور والإدارة، بالإضافة إلى العلاقات بين هذه الكيانات داخل قاعدة البيانات.

Schema

يوضح الـ Schema الشكل الفعلي لهيكل قاعدة البيانات، بما يشمل الجداول والحقول الأساسية والمفاتيح المستخدمة لربط البيانات وتنظيمها داخل النظام.

UI / UX Design

تصميم الواجهات

Screen 01

Home Page

توضح الصفحة الرئيسية واجهة ترحيبية حديثة تعرض فكرة النظام، مثل الخدمات الأساسية.

Screen 02

Sign Up Page

توضح شاشة إنشاء الحساب إدخال بيانات المستخدم وتحديد نوعه، مع حفظ البيانات داخل Firebase وربطها بصلاحيات النظام المختلفة.

Screen 03

Login Page

توضح شاشة تسجيل الدخول آلية دخول المستخدمين إلى النظام بناءً على الأدوار المختلفة مثل الطالب، المعلم، ولي الأمر، والأدمن.

Screen 04

Student Dashboard

توضح لوحة تحكم الطالب البيانات الأكاديمية الخاصة به، مثل المواد الدراسية، الدرجات، نسبة الحضور، والإشعارات، مع تصميم سهل يساعد الطالب على متابعة مستواه الدراسي.

Screen 05

Admin Dashboard & AI Attendance Control

توضح لوحة تحكم الأدمن التحكم الكامل في نظام الحضور الذكي، حيث يمكن بدء وإيقاف الكاميرا، متابعة حالة النظام، عرض نسبة التعرف، ومراقبة عملية تسجيل الحضور داخل النظام بشكل مباشر.

Frontend

الواجهة الأمامية

Sign Up

1

يتم إنشاء حساب جديد للمستخدم من خلال Firebase Authentication باستخدام البريد الإلكتروني وكلمة المرور.

2

بعد إنشاء الحساب يتم حفظ جميع بيانات الطالب داخل Firestore مع تحديد الدور الخاص به.

3

يتم رفع صورة الوجه وربطها ببيانات الطالب لاستخدامها لاحقًا في نظام الحضور الذكي.

Login

1

يتم التحقق من البريد الإلكتروني وكلمة المرور باستخدام Firebase Authentication لضمان دخول آمن للمستخدم.

2

بعد نجاح تسجيل الدخول يتم إنشاء Token خاص بالمستخدم وحفظه لاستخدامه في الوصول الآمن للـ APIs.

3

يتم تحديد دور المستخدم وتحويله تلقائيًا إلى لوحة التحكم الخاصة به.

Backend

الخادم

Firebase Token Verification

يوضح هذا الجزء آلية التحقق من الـ Firebase ID Token داخل Laravel Middleware، للتأكد من هوية المستخدم قبل السماح له بالوصول إلى الـ API.

Protected API Routes

يوضح هذا الجزء حماية الـ API Routes باستخدام Middleware، بحيث لا يمكن الوصول إليها إلا بعد التحقق من الـ Token بنجاح.

Validation

يوضح هذا الجزء التحقق من صحة البيانات المدخلة قبل تنفيذ أي عملية، وذلك لمنع إدخال قيم غير صحيحة أو غير مكتملة إلى النظام.

Artificial Intelligence

الذكاء الاصطناعي

نظام التعرف على الوجه وتسجيل الحضور التلقائي

01

Face Recognition Model Initialization

يوضح هذا الجزء تهيئة نموذج التعرف على الوجه باستخدام InsightFace، وتجهيزه لاكتشاف الوجوه واستخراج خصائصها.

02

تحميل بيانات الوجوه من Firebase

يوضح هذا الجزء تحميل صور الطلاب من Firebase، ثم استخراج السمات المميزة لكل وجه وتحويلها إلى Embeddings لاستخدامها في عملية التعرف.

03

مطابقة الوجوه والتعرف على الطالب

يوضح هذا الجزء مقارنة الوجه المكتشف من الكاميرا مع الوجوه المخزنة مسبقًا، ثم تحديد هوية الطالب بناءً على أعلى نسبة تشابه.

تسجيل الحضور تلقائيًا

يوضح هذا الجزء تسجيل حضور الطالب تلقائيًا داخل Firebase بعد نجاح عملية التعرف على الوجه، مع حفظ التاريخ والوقت وحالة الحضور.
